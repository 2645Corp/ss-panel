0x01: 起源简介

首先简单介绍一下什么这一系列东西的起源和历史。

在很久很久以前，我们访问各种网站都是简单而直接的，用户的请求通过互联网发送到服务提供方，服务提供方直接将信息反馈给用户 

GFW

GFW 是 Great Fire Wall 的简称，即超级防火墙或防火长城。中国所有的出国流量全部要经过 GFW。GFW 会监视你发送的内容，将它不喜欢的内容统统过滤掉，于是客户当触发 GFW 的过滤规则的时候，就会收到 Connection Reset 这样的响应内容，而无法接收到正常的内容。

SSL/TLS

过去 HTTP 协议中，传输的内容是明文的。即，GFW 可以知道你向服务器发送了什么内容，以及服务器返回了什么内容。这种情况下，GFW 可以选择性地屏蔽某些“不符合中国审查规则”的内容。出于安全考虑，依照现在的互联网规范，绝大多数的网站都已经强制 HTTPS。当你访问一个网站时，看到地址栏上出现绿色挂锁图标时，你的连接就已经是 HTTPS 连接。此时发送的数据是 SSL/TLS 加密的，你向网站发送敏感信息（如密码）时，只有网站能够解密，运营商（如中国电信）和 GFW 都无法解密。 因此 GFW 看不到网站发送的内容，而某些网站（Google/Twitter/Facebook）又拒绝配合中国政府的审查，于是 GFW 选择直接屏蔽掉这些网站的全部连接。

VPN/SSH tunnel

聪明的人们想到了利用境外服务器代理的方法来绕过 GFW 的过滤，其中包含了 VPN 服务和 ssh tunnel 服务，以 ssh tunnel 为例：

1) 首先用户和境外服务器基于 ssh 建立起一条加密的通道 2-3) 用户通过建立起的隧道进行代理，通过 ssh server 向真实的服务发起请求 4-5) 服务通过 ssh server，再通过创建好的隧道返回给用户

由于 SSH 本身就是基于 RSA 加密技术，所以 GFW 无法解密传输数据。而发送的目标是 SSH Server，真实的目标地址（Google/Twitter/Facebook）是隐藏在加密数据中的，只有 SSH Server 能够解密，这样就可以绕过 GFW 限制访问它屏蔽的网站。

缺陷：VPN 和 SSH 本身并不是专门为翻墙设计的东西，他们能够很好的保证安全：即传输过程中数据的保密性和完整性，以及对传输双方身份的校验。但是由于协议特征过于明显，导致 GFW 很容易识别：啊，这是 VPN！没备案！切断！GG！

Shadowsocks

2015 年，clowwindy 同学创作并开源了他的解决方案。

简单理解的话，shadowsocks 是将原来 ssh 创建的 Socks5 协议拆开成 server 端和 client 端，所以下面这个原理图基本上和利用 ssh tunnel 大致类似：

与 SSH 和 VPN 显著的区别是，这个传输过程中没有明显的特征。SS Local 和 SS Server 之间的通信就是普通的 TCP 连接，数据是加密的 GFW 也无法解密数据，因此 GFW 无法像识别 VPN 那样轻易地识别这是一个“翻墙”连接。

ShadowsocksR

shadowsocksr 是 shadowsocks 的一个改进版本。为了对抗 GFW 的深度包检测，breakwa11 对 shadowsocks 进行了改进：即除了 shadowsocks 协议（shadowsockr 中称为 origin 协议）以外，还发布了若干新协议，并且定期会发布新的协议。新的协议改进旧协议的缺陷，使得更难被 GFW 检测。

shadowsocksr 还支持混淆，即将加密流量伪装成正常的 https 网站加密流量，从而在有 QoS 的 网络环境中提速。

shadowsocks 方面则认为没有依据证明 GFW 是依据这些方法检测 shadowsocks 的（当然到目前为止没有人能够证明 GFW 的检测方法，只能靠体验来判断哪个比较好用），针对这些方法的改进可能会暴露其他的弱点，因此不接受这些改进意见。而混淆的部分，shadowsocks 则是通过单独的混淆插件来提供，没有纳入到 shadowsocks 核心程序中。

关于 shadowsocksr 的争议和讨论一直持续到 2017 年 7 月。由于 shadowsocksr 一直宣称比 shadowsocks 好，以及散布 shadowsocks 很快就要被识别就不能用了等言论，breakwa11 在 2017 年 7 月遭到人肉攻击，从此放弃了 shadowsocksr 的开发和维护。

ShadowsocksRR

R 表示革新（revolution），shadowsocksrr 是 breakwa11 离开后由 Akkariiin 维护的分支。它在 shadowsocksr 的基础上又发布了很多新协议。

V2Ray(VMess)

2017 年公开的 Project V 发布了新的协议 VMess，VMess 协议与 shadowsocks 协议相比，也包括一些针对深度包检测的改进。

V2Ray 是比 shadowsocks 系列更加强大的工具。不同于 shadowsocksr 的伪装路线，V2Ray 能够将流量隐藏在真实的 HTTP 或 WebSocket 中，支持真实、完整的 TLS。

2：启动服务

客户端分为电脑客户端和手机客户端。两者的主要区别是电脑客户端激活的是 socks5 代理，而手机客户端激活的是 vpn。

手机客户端一般都有一个连接的按钮，点击一下就可以启动 vpn。

电脑客户端服务会自动启动。只要勾上托盘菜单 -> 服务器中的相应服务器，socks5 代理就已经激活。

※ v2rayN 初次使用需要先点击检查更新 v2rayCore：

当看到“[Warning] Core: V2Ray v3.29 started”时表示 v2ray 已经启动并激活 socks5 代理。

3：配置网络

我们需要配置网络使得我们的网络经过 shadowsocks、v2ray。

对于手机客户端，激活系统 vpn 后所有网络流量都会通过 shadowsocks、v2ray。

对于电脑客户端，激活 socks5 代理后不会有任何网络流量通过 shadowsocks、v2ray，你还需要配置网络代理。

1. 系统代理：

Windows 客户端的托盘菜单里有“启动系统代理”。点击以后客户端将会帮你启动 IE 代理。请注意，IE 代理并不是全局系统代理，只有跟随 IE 代理的应用程序才能够被代理。通常情况下浏览器是会跟随 IE 代理的，而有些程序可以设置跟随 IE 代理。

2. 手动设置代理：

很多程序都可以手动设置代理。例如上面的 QQ，将类型选择 socks5 代理，配置地址为 127.0.0.1（本地回环地址），端口为本地监听端口（local_port）（通常为 1080）。

3. 浏览器插件：

在不开启系统代理的情况下，可以通过浏览器插件来使浏览器通过代理。

在扩展商店中下载 SwitchyOmega。

创建一个情景模式“代理服务器”，设置代理协议 socks5，代理服务器 127.0.0.1，代理端口 1080：

点击插件图标切换到刚才创建的情景模式即可：

4. 全局代理

要让所有的应用程序通过 socks5 代理上网，一种方法是通过 Proxyfier 进行代理。如果你要使用可能需要为购买 Proxyfier 支付一定的费用。

点击 Profile->ProxyServers，添加一个代理服务器，设置地址为 127.0.0.1，端口 1080，协议 socks5：

点击 Profile->Proxification Rules，设置除了本地地址以外都通过上面添加的 server 代理。

设置完成后你可以保存当前配置。这样你可以保存多个配置来方便在不同配置中切换。例如，你可以配置“除了本地地址都通过代理”和“全部直连”，然后可以在托盘菜单中方便地切换它们。



※ 如果你是 Windows8、Windows8.1、Windows10 用户，需要通过代理连接使用 Metro App、Windows Modern App、UWP App 的话，由于系统默认将他们在沙箱中运行，你需要先将他们解锁。一种解锁方法是使用 fiddler 抓包工具：

点击 WinConfig，然后勾上想要解锁的 App 即可。

4：路由

手机客户端：

设置路由可以选择哪些地址通过代理。设置分应用代理可以选择哪些应用通过代理。

SwitchyOmega：

创建情景模式“自动切换模式”，规则列表格式选择 AutoProxy，在规则列表网址中填入 https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt

gfwlist 是一个项目，它维护着所有已知的被 GFW 屏蔽的网站。选择这个情景模式即为 “仅代理中国大陆无法访问的地址”。
